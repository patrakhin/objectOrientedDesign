Рефлексия задания 4.
Незначительные расхождения с эталоном, тем не менее,
не определено, когда делать модуль открытым, а когда закрытым.
(По примеру модуля отправки документов внешнему провайдеру)

Решение задания 5.
1.	Можно использовать обобщения (generics), чтобы создавать классы, интерфейсы и методы, которые работают с параметризованными типами.
2.	Можно объединять функциональность внутри класса, создавая методы, которые взаимодействуют друг с другом.
3.	Можно организовывать классы в пакеты, создавая семейства модулей для решения общих задач.
4.	Можно использовать наследование и переопределение методов для предоставления конкретной реализации в подклассах.
5.	Можно использовать интерфейсы и наследование, чтобы интегрировать общее поведение для различных классов.
Но, понятие модуля в Java не определено. Поэтому, вынуждены не просто применять принципы а выражать их через инструменты в Java.
